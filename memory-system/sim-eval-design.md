# Vex Memory — Time-Simulated End-to-End Correctness Eval (Design, 2026-06-13)

Designed on Opus 4.8 (3 grounding agents + synthesis). Purpose: BEFORE any further
memory change, run a fictional agent's 90-simulated-day stream (trades / regime / lessons,
produced by a subagent) through the REAL pipeline (suggest → live DeepSeek judge →
consolidation/graph/decay/reconcile → Gemma retrieval) ONE ITEM AT A TIME as if live, and
score actual memory state against a PRE-REGISTERED INDEPENDENT ORACLE. This is the
independent-oracle + emergent-behavior axis the adversarial review (test-plan-adversarial-
review.md) proved missing.

> valid-rate ≈0%". That is STALE — Batch F (commit 269560f) fixed F31; the live eval

## ARCHITECTURE
DECISION: one new self-contained eval suite, `src/__tests__/integration/eval/e2e-memory-correctness.int.test.ts`, plus three new sidecar modules (`_world-corpus.ts`, `_oracle.ts`, `_sim-clock.ts`), reusing the existing harness wholesale (vitest/eval.config.ts globalSetup → testcontainers pg + real Gemma probe + OpenRouter key; _eval-fixtures.ts drivers; _report-card.ts disk-backed collector). It runs ONE describe.skipIf(!OPENROUTER_API_KEY) with ordered `it()` per simulated checkpoint, and feeds the same reportCard singleton zz-report.int.test.ts already flushes.

Shape (functional core / imperative shell, per rules/10 §3): a 90-day stream of ~140 events (100 memory items + ~30 trades + ~10 regime snapshots) is authored at module load as PURE data in `_world-corpus.ts`, each tagged with a simulated ISO date (`simDay: 0..89`). The oracle (`_oracle.ts`) is a SEPARATE pure data file: a `Map<itemId, OraclePrediction>` plus a `RetrievalOracle[]` of queries with `expectedTopIds`/`mustNotAppearIds`. The runner is the imperative shell: a single `runStreamItem(event, simNow)` that dispatches by event kind to the REAL pipeline door (handleLongMemorySuggest for memory items, seedFaithful* for trades, insertRegimeSnapshot for regimes), then for memory items drives the REAL judge via driveConsolidateCapturingJudge, advancing a simulated clock between checkpoints.

The end-to-end loop is exactly the live runtime path, never a shortcut: handleLongMemorySuggest (real redact/scanLiveState/english-check/loop-prevention/embed/insertCandidate/enqueueConsolidateJob, suggest.ts:183-394) → driveConsolidateWithRealJudge (enqueue→claim→reserve→consolidateCandidate with defaultConsolidateDeps → live DeepSeek judge → applyDecisionAtomically with full S5/S8 arg shape, _eval-fixtures.ts:569-628) → on trade-close events seedFaithfulClosingTradeForWake fires enqueueLedgerWake → claimNextDueJob + processReconcileJob (reconcile.ts:203) → between checkpoints runDecaySweep(SIM_NOW) (decay-sweep.ts:90) after backdating last_decayed_at. After the full stream, a SNAPSHOT phase reads final memory state via the real read paths and a SCORING phase compares to the pre-registered oracle, emitting hard-gate expect() failures and scored reportCard rows.

A 100-item × live-judge run is the heaviest eval; it is gated behind its own env flag `VEX_E2E_MEMORY_EVAL=1` (mirroring the existing VEX_REAL_DENSE_EVAL pattern in package.json) and a new `test:eval:e2e` script, so the cheap per-finding evals stay the default `test:eval` run. testTimeout already 240s; the e2e file budgets ~100 judge round-trips (tens of seconds each) so it carries its own longer per-it timeouts and runs serialized (fileParallelism:false already set).

## TIMESIMULATION
There is NO global clock seam (confirmed: crud.ts:271-272 NOW(), promote.ts:181 new Date(), recall.ts:56 / crud.ts:606 / hot-context.ts:56 now(), consolidate.ts:482 calls runDeterministicStage WITHOUT a `now` so D10 TTL uses real new Date() at deterministic-stage.ts:150). So time simulation is TWO-LEVER per the feasibility split: (A) inject SIM_NOW into the pure-function elapsed-time math, and (B) backdate the row timestamps that real-clock reads diff against, atomically, at each checkpoint.

CONCRETE LEVER PER TEMPORAL BEHAVIOR:
- DECAY (CLASS A inject): call `runDecaySweep(SIM_NOW)` directly (decay-sweep.ts:90 takes injected now; the executor's bare runDecaySweep() at executor.ts:160 is NOT used). The anchor is laterOf(lastReinforcedAt||firstPromotedAt, lastDecayedAt) (maturity.ts:285). CRITICAL compounding fix (feasibility risk #1, HIGH): an applied decay stamps last_decayed_at=NOW() (crud.ts:272), so BEFORE each multi-day advance, backdate last_decayed_at to the PRIOR SIM_NOW via UPDATE — else the next sweep sees Δt≈0 and below_delta no-ops forever. Also backdate first_promoted_at (promote.ts:181 not controllable at write, so UPDATE it post-promote to its simDay) so daysSince (maturity-policy.ts:321) yields the intended age. To actually reach the `decayed` tier (activation ≤ 0.2 at threshold, half-life 30d) seed at activation 1.0 and advance ~90 sim-days with intermediate sweeps; closed-form: activationAfter = max(0.03, 1.0·0.5^(days/30)).
- REGIME decay modulation (CLASS A inject + backdate): insertRegimeSnapshot has NO created_at param (always NOW(), regime-snapshots.ts), so insert the two dwell snapshots then UPDATE created_at to two consecutive sim-days (gap ≤ 48h, REGIME_DWELL_MAX_GAP_HOURS); pass effectiveRegime(getLatestTwoRegimeSnapshots(), SIM_NOW) into the sweep via a custom DecaySweepDeps.getEffectiveRegime that closes over SIM_NOW (feasibility risk #3 — the default deps use new Date() at decay-sweep.ts:72). Newest snapshot must be within 3d of SIM_NOW (REGIME_SNAPSHOT_MAX_AGE_DAYS) or it fails closed to null. Reactivation needs HIGH confidence in BOTH snapshots.
- RETRIEVAL TTL / dual-trace expiry (CLASS B backdate): candidate recall filters retrieval_until > now() (crud.ts:606); knowledge recall + hot-context filter valid_until (recall.ts:56, hot-context.ts:56). UPDATE retrieval_until / valid_until to a past sim-date to expire. This is the F3 lever (slow-recurring lesson: backdate the first observation's retrieval_until past the 7d window so recurrence stays 1).
- RECONCILE / outcome flip (NO time dependence — outcome-resolver.ts is ledger-only, point-in-time.ts derives boundary from eventTime): drive a real later closing trade (seedFaithfulClosingTradeForWake) whose instrumentKey matches the promoted lesson's persisted evidence_refs so enqueueLedgerWake (capture-pipeline.ts) fires; the stored available_at_decision_time (persisted at promote) governs pointInTimeChecked (reconcile.ts:268). For a deterministic flip, store a positive old outcome (updateCandidateOutcome) and seed the underlying ledger as a loss; outcome_version on stored candidate MUST equal entry.outcome_version (both 0) or reconcile no-ops (feasibility risk #6, reconcile.ts:244).
- PROMOTE world-time validity: validFrom is controllable (consolidate.ts:665 via outcome boundary / seedPromotedLessonDirect.validFrom); set it to the item's simDay.

ATOMICITY DISCIPLINE (feasibility risk #5): a `backdateRow(entryId, simDate)` helper does ONE UPDATE setting created_at, first_promoted_at, last_reinforced_at, last_decayed_at, retrieval_until/valid_until consistently to the same sim-anchored values, so no read ever sees a mixed real/sim timeline. `_sim-clock.ts` owns SIM_EPOCH (a fixed past date, e.g. 2026-03-01) and `simNow(day) = SIM_EPOCH + day·86400s`.

Author-1 writes all 100 with rationale; Author-2 (a second pass, ideally a different model/session) blindly re-derives ≥30% and any disagreement is logged as `oracle_dispute` — forcing a human reconciliation before the number is trusted.

## ANTICIRCULARITY
The single deepest requirement (test-plan-adversarial-review.md: the suite must detect WRONGNESS, not just DRIFT). Five disciplines:

1. ORACLE AUTHORED FROM PRODUCT INTENT, NOT CODE CONSTANTS. The oracle never imports clampSourceTier, consequenceFor, deriveEvidenceStrengthCeiling, DECAY_FLOOR, NEAR_DUP_COSINE, or any policy module. Expected verdicts and the decay/quench/floor numbers are hand-typed literals with a prose rationale per item. A pipeline-vs-oracle disagreement is triaged as `memory_bug` OR `oracle_error` (a human decides) — it is NOT auto-resolved by trusting either side. This is what makes the BROKEN-DECAY / MIS-TUNED-DEWEIGHT / STEERED-JUDGE escapes from the review visible: e.g. an oracle hardcoded "INFERRED item X must NOT outrank OBSERVED item Y" reds if SOURCE_SOFT_WEIGHT is shipped at 0.99, because the oracle's expected order is independent of the constant.

2. TWO-AUTHOR + COMMITTED HASH. Oracle committed before the run (ORACLE_GIT_HASH asserted); ≥30% blind-re-derived by a second author; disputes block trust. Prevents over-fitting the oracle to one author's reading of the code (world-corpus risk #7).

3. HARD vs SCORED SEPARATION IS THE FIREWALL. Only SPEC-derivable structural invariants are expect()-gated (a superseded row is not active → not recalled; a reject writes no row; floor 0.03 is a hard physical bound). Everything the MODEL decides (promote-or-not, graph content, supersede target choice, tier raise) is SCORED via reportCard, never asserted — so live-judge nondeterminism (F31) can never turn the green/red signal into noise, AND a green run never falsely implies "the judge judged correctly." The report-card sinks are pure push+persist (no expect/throw, confirmed _report-card.ts:177-231), so the measured lane cannot mask a hard-gate failure.

4. COUNT ONLY VALIDATED ITEMS for the model-quality metrics. Promotion-precision denominator = items where verdictValid (F31 valid verdicts), so a 0%-valid-rate judge yields "unmeasured", not a misleading 100% or 0%.

5. NO SHORTCUT FAITHFULNESS. Every entry passes through the real door or real insertEntry with REAL Gemma dim-768 embeddings (never dim-8 fixtures — recall.ts:34 throws on dim mismatch and returns nothing on model mismatch, so a wrong embedding silently zeroes recall: pipeline risk #2). The judge is the real DeepSeek; the ledger is real proj_pnl rows; reconcile fires off a real capture-seam wake. The ONLY short-circuit is seedPromotedLessonDirect for deterministic predecessors/targets, and it uses the production insertEntry path — only the judge verdict is bypassed, and only where the oracle is scoring DOWNSTREAM behavior (supersede/reconcile/recall), never the promote decision itself.
